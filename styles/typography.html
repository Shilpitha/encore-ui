• ENCORE-SPECIFIC: We use Google Fonts, in particular the Roboto font. While Google Fonts are not included in the Encore UI package, they are included if you use the encore-ui template repository as a base.
  • For developers not using encore-ui-template, read the README.md file for rxApp on fonts for more information.

Headings/Title Styles

  • Headings are sized according to their heading level (H1 larger than H2 larger than H3, etc.).
  • ENCORE-SPECIFIC: For design consistency, we usually use an H3 heading for most subheaders. Other headings can be used for context.
  • WARNING: The title, xl, lg, sm, and xs heading classes are deprecated and will be removed in a future release of the framework.

Descriptions / Metadata

  • The rxMetadata component provides the ability to display a list of terms with their associated descriptions. This is used throughout Encore to show metadata, especially on an individual object such as a server or a CBS volume.
  • ENCORE-SPECIFIC: This is an ideal solution if you have anywhere from 5-15 definitions (3 columns of 5 definitions). (See a Server Details page or Image Details page under Accounts -> Cloud for an example.) Anything more - or anything displaying visual or image content - and you may want to consider a different design pattern. (See the "Non-Data based Tables" section in the Table page for more information.)

When There is No Data to Display

  • In table cells, metadata lists, or other displays of data, N/A should be used as an indicator to the user that there is no relevant data to display.
  • Do not use â€“, Empty, null, or no indicator to communicate this, as there may be situations where those are valid inputs or data to display to a user depending on the context.
  • Note that a separate style exists to communicate when data isn't present due to permissions , or when an entire table is empty (see "Null Patterns" section of Tables style guide ).

Lists

  • Lists by default do not have styles. Add bullets/decimals to lists with the list class. It is also possible to build non-semantic lists using the additional item class.
  • ENCORE-SPECIFIC: Use this in conjunction with icons, colors and helper classes such as .flush-right to build out action menus. Look at the detail pages in Cloud Servers and view-source for additional context.
